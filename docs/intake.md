# Project Intake — BeerRank

## A1. 產品定義

### 產品定義
- 一句話定義：BeerRank 是一個給啤酒愛好者用照片記錄品飲、發表酒評、互相按讚，並累積啤酒排行榜的社群 Web App。
- 目標使用者：啤酒愛好者、精釀酒吧常客、想記錄喝過啤酒的人、想找推薦酒款的人。
- 核心價值：把「像 IG 一樣分享品飲照片」和「結構化啤酒評分、排行」結合，讓個人紀錄與群體推薦可以互相累積。
- 產品階段：全新未驗證。
- 商業模式：TBD；初期以個人專案與社群驗證為主。

## A4. 第一個要做的功能

### 第一個功能
- 功能名稱：社群酒評貼文與啤酒排行榜 MVP。
- 想解決的問題：使用者可以拍照分享啤酒，但一般社群平台缺少啤酒專屬欄位、個人評分與可累積的排行榜。
- 使用者場景：使用者在喝到一款啤酒時，用手機上傳照片、選擇或新增啤酒、給出評分和短評，發佈後其他人可以瀏覽、按讚，系統可依評分與互動產生排行榜。
- 期望結果：使用者能完成第三方登入、建立酒評貼文、看到動態牆、按讚貼文、查看啤酒排行榜。
- 明確不做：私訊、好友系統、酒吧/酒廠商家後台、進階推薦演算法、付款、完整內容審核系統、原生 App。
- 資料概念：使用者、第三方登入身份、啤酒、酒廠、酒評貼文、照片、評分、按讚、排行榜統計。
- 已知限制：先做 Web App MVP；排行榜演算法先用簡單可解釋規則；圖片上傳先限制單張。
